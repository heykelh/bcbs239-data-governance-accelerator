# Data Lifecycle

## Lifecycle stages

1. Data creation in source systems
2. Ingestion into the central platform
3. Transformation and standardization
4. Data quality controls
5. Consumption in risk reporting
6. Archiving and retention

## Key control points

- ingestion completeness
- mandatory field presence
- format validity
- duplicate detection
- date coherence
- business rule conformity

## Governance implication

Each stage should have:
- a control owner
- a monitoring KPI
- a remediation workflow

## Why it matters

A data governance model is only effective if it is connected to the actual lifecycle of the data.  
This ensures that ownership and controls are not theoretical, but attached to specific operational stages.
