# Gap Analysis

## Initial situation

The simulated bank faces several common weaknesses around critical risk data:

- no formal ownership per domain
- inconsistent quality controls
- limited visibility on data incidents
- no shared KPI framework
- fragmented responsibilities across business and IT

## Gap analysis

| Area | Current state | Target state | Recommendation |
|---|---|---|---|
| Ownership | Informal and fragmented | Formal ownership per business domain | Nominate Data Owners and validate accountability |
| Data quality controls | Ad hoc and heterogeneous | Standardized control library | Define reusable DQ rules |
| Monitoring | Limited visibility | KPI-based steering | Publish a recurring DQ dashboard |
| Metadata | Partial and inconsistent | Shared glossary and dictionary | Maintain critical data elements repository |
| Issue management | Reactive | Structured remediation workflow | Track incidents with severity and SLA |

## Priority actions

1. Formalize governance roles
2. Define critical data elements
3. Implement core DQ rules
4. Create KPI dashboard
5. Establish a recurring governance review

## Expected outcome

The target state improves:
- ownership clarity
- issue detection
- remediation follow-up
- management visibility
- trust in critical reporting data
